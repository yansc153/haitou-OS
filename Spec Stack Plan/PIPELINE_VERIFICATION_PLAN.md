## 零、核心验证原则

### ⚠️ 必须遵守的规则

1. **所有验证必须通过前端 MCP Playwright 进行**
   - 打开页面 → 截图 → 看用户能看到什么 → 判断是否正确
   - 禁止用 curl/REST API 查数据库来判断"通过"
   - 如果前端看不到，等于不存在

2. **验证流程：看 → 判断 → 推导**
   - 第一步：在浏览器里看到了什么？（截图）
   - 第二步：看到的内容是否符合预期？
   - 第三步：如果不符合，再去查后端日志/数据库找根因
   - 禁止跳过第一步直接查数据库

3. **每一步都要截图**
   - 命名规则：`e2e/pipeline-{phase}-{step}-{描述}.png`
   - 截图是验证证据，不是装饰

4. **前端没有反馈 = bug**
   - 如果后端做了事但前端没展示 → 前端 bug，记录并修复
   - 如果前端显示错误信息 → 记录错误文案，判断是否准确
   - 如果前端空白无反馈 → 最严重的 bug，必须修

### 验证工具链

| 工具 | 用途 | 何时使用 |
|------|------|---------|
| MCP browser_navigate | 打开页面 | 每个 Phase 开始 |
| MCP browser_snapshot | 读页面结构 | 需要点击元素时 |
| MCP browser_take_screenshot | 截图存证 | 每个验证点 |
| MCP browser_click | 点击操作 | 模拟用户行为 |
| MCP browser_wait_for | 等待加载 | 等 Worker 处理 |
| fly logs | 查 Worker 日志 | **仅在前端发现问题后** 用于诊断根因 |
| curl REST API | 查数据库 | **仅在前端发现问题后** 用于诊断根因 |

## 一、三种投递模式定义

### Mode 1: 英文定制投递 (full_tailored)
```
平台: Greenhouse, Lever
流程: 发现 → 筛选 → 定制简历 → Cover Letter → 表单投递
特点: 每个岗位单独生成材料，通过 Playwright 填表提交
回复: 不追踪（投递用 email，回复在邮箱）
```

### Mode 2: 中文批量直投 (passthrough)
```
平台: 智联招聘, 拉勾, 猎聘
流程: 发现 → 筛选 → 直接投递（用 onboarding 优化过的简历）
特点: 不改简历，批量投递，速度优先
回复: 不追踪（平台站内信通知）
```

### Mode 3: Boss 直聘对话式 (passthrough + conversation)
```
平台: Boss直聘
流程: 发现 → 筛选 → 投递 → 发打招呼消息 → 等回复 → 聊几句 → 检测面试信号 → 通知用户
特点: 唯一需要对话的平台，聊天内容需要 AI 理解
回复: 主动追踪，检测"约面试"等关键信号
```

## 四、测试执行方案

### 准备工作

```
1. 确认测试用户 plan = plus
   → 前端验证: 首页右下角 "当前使用 plus 方案"
   
2. 确认所有平台已连接
   → 前端验证: 平台中心 7 个平台都显示 "已连接"
   → 如果有 cookie 平台未连接: 用户手动断开重连

3. 确认 Worker 运行中
   → 前端验证: 首页右上角 "运行中" 绿色指示
   → 首页团队阵容 7 个 Agent 都 "运行中"
```

### Scenario 1: Mode 1 — Greenhouse/Lever 英文定制投递

```
步骤 1: 打开首页 → 截图 → 确认 Worker 运行中
步骤 2: 等待 2-5 分钟 → 刷新首页 → 截图实时动态
步骤 3: 打开机会中心 → 截图岗位列表 → 确认有新岗位
步骤 4: 点击一个 Greenhouse 岗位 → 截图详情
步骤 5: 展开 JD → 截图 → 人工阅读判断质量
步骤 6: 看 AI 评估（匹配度+建议）→ 截图
步骤 7: 如果有材料 → 截图简历对比视图 + Cover Letter
步骤 8: 如果有投递 → 截图投递状态
步骤 9: 回到首页 → 截图今日运营统计

如果某一步看不到预期内容:
  → 记录"前端缺失: [描述]"
  → 然后查 fly logs 诊断根因
  → 记录"根因: [描述]"
  → 创建修复任务
```

### Scenario 2: Mode 2 — 中文批量直投

```
步骤 1: 打开机会中心 → 找中文岗位（字节跳动/美团等）
步骤 2: 点击中文岗位 → 截图详情
步骤 3: 确认 JD 是中文 → 截图
步骤 4: 确认没有"AI 定制简历"区域（passthrough 模式）
步骤 5: 看投递状态 → 截图
步骤 6: 回到首页 → 确认实时动态有中文投递事件

如果看不到中文岗位:
  → 平台中心确认智联/拉勾/猎聘已连接
  → fly logs 查 cookie 解密是否成功
  → 如果 cookie 失败 → 用户需要重连平台
```

### Scenario 3: Mode 3 — Boss 直聘对话式

```
步骤 1: 机会中心 → 找 Boss 直聘的岗位
步骤 2: 点击 → 看打招呼消息内容 → 截图
步骤 3: 确认阶段变为"联系"
步骤 4: （模拟对方回复 — 手动在 Boss 上回消息）
步骤 5: 等 15 分钟 → 刷新 → 看是否检测到回复
步骤 6: 如果检测到面试信号 → 看交接中心
```

### Scenario 4: Loop B — 跟进 + 交接

```
步骤 1: 找一个 3 天前投递但无回复的 Boss 岗位
步骤 2: 等待 follow_up sweep 触发
步骤 3: 首页实时动态 → 看跟进消息事件
步骤 4: 机会中心 → 该岗位 → 看跟进记录
步骤 5: 模拟"可以约个面试"回复
步骤 6: 交接中心 → 看是否出现待处理项
步骤 7: 首页 → 看"需要你接管"卡片
```

## 五、质量检查清单（全部在前端看）

### AI 输出质量

| 检查项 | 在哪看 | 标准 |
|--------|--------|------|

| 交接中心 | 有交接时显示，无交接时"一切顺利" |
| 活动回顾统计 | 数字和实际操作一致 |
